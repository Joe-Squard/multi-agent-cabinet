# /review-now — ローカル差分の独立コードレビュー

ワーキングディレクトリの変更差分をレビューします。

## 使い方

```
/review-now [path]          # 指定パスの変更をレビュー
/review-now --staged        # ステージ済みの変更をレビュー
/review-now --branch <name> # ブランチの差分をレビュー
```

## レビュー観点

1. **セキュリティ**: XSS, SQLi, 認証/認可, シークレット露出
2. **バグ**: 論理エラー, エッジケース, null/undefined, 型不一致
3. **パフォーマンス**: N+1, 不要な再レンダリング, メモリリーク
4. **可読性**: 命名, 複雑度, マジックナンバー
5. **スコープ**: リリーススペック内の変更のみか

## 実行手順

1. 差分を取得:
```bash
# ステージ済み
git diff --staged
# ブランチ差分
git diff develop..HEAD
# ワーキングディレクトリ
git diff
```

2. 差分を分析してレビューコメントを生成
3. 重要度別に整理:
   - **CRITICAL**: 即修正必要（セキュリティ、データ損失）
   - **WARNING**: 修正推奨（バグの可能性、パフォーマンス）
   - **INFO**: 改善提案（可読性、スタイル）

4. 結果をフォーマットして表示
