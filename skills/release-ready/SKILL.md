# /release-ready — リリース前セルフ評価チェックリスト

実装完了後のセルフレビューチェックリストを実行します。

## 使い方

```
/release-ready [branch_name]
```

## チェックリスト

### 1. スコープ確認
- [ ] リリーススペックのスコープ内の変更のみか
- [ ] スコープ外の変更がないか
- [ ] 不要なファイルが含まれていないか

### 2. テスト
- [ ] 全テストが通るか (`npm test` / `pytest` 等)
- [ ] 新規コードにテストがあるか
- [ ] エッジケースのテストがあるか

### 3. コード品質
- [ ] Conventional Commit 形式になっているか
- [ ] マジックナンバーがないか
- [ ] エラーハンドリングが適切か
- [ ] 不要な console.log / print がないか

### 4. セキュリティ
- [ ] ハードコードされたシークレットがないか
- [ ] ユーザー入力のサニタイズがされているか
- [ ] 認証/認可のチェックがあるか

### 5. ドキュメント
- [ ] 破壊的変更がある場合、記載されているか
- [ ] 公開 API の変更がある場合、ドキュメントが更新されているか

## 実行手順

1. ブランチの差分を取得（develop からの差分）
2. 上記チェックリストを自動 + 手動で評価
3. 結果をサマリーで表示
4. Growth: 推奨（スキップ可）/ Maintenance: 必須（スキップ不可）

```bash
# 差分取得
cd <worktree_path>
git diff develop..HEAD --stat
git diff develop..HEAD
```
